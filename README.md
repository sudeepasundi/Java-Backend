# Backend Development Roadmap Notes

This workspace is organized as a topic-wise backend development learning path from Java basics to advanced distributed systems.

## How to Use These Notes

1. Read each numbered folder in order.
2. Run the code snippets locally and modify them.
3. After each module, build a small project that combines the topics.
4. Revisit later modules after building real APIs, because topics like security, caching, messaging, and observability become clearer with practice.

## Folder Map

| Module | Folder | Focus |
| --- | --- | --- |
| 1 | `01-java-fundamentals` | Java syntax, OOP, collections, JVM, threading, web basics, tooling |
| 2 | `02-spring-framework` | IoC, DI, bean lifecycle, MVC, profiles, AOP |
| 3 | `03-spring-boot` | Boot advantages, auto-configuration, properties, integrations |
| 4 | `04-rest-api-development` | HTTP methods, status codes, REST design |
| 5 | `05-databases` | SQL, NoSQL, Spring Data JPA, MongoDB, JDBC |
| 6 | `06-spring-security` | Authentication, authorization, OAuth2, form login, JWT |
| 7 | `07-microservices` | Spring Cloud, Docker, Kubernetes, cloud, patterns, messaging |
| 8 | `08-software-engineering-backend` | Clean code, system design, scaling, caching, observability |

| Spring Boot REST | Task manager API | REST, JPA, DTOs, error handling |

| Microservices | Order platform | Gateway, config, circuit breaker, messaging |

## Learning Rule

Do not only read the notes. Every concept should be converted into:

- one small code example,
- one test,
- one diagram,
- one explanation in your own words.
